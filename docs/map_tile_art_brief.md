# 大地图格网 · 美术资源需求说明（AI 生成用）

本文档基于当前 **`web/explorer/main.js`** 中的实现整理，供后续 **AI 绘图 / 像素_tile** 或 **程序贴图** 使用。仓库内 **尚无** 独立地图贴图文件，主地图为 **程序色块 + 矢量叠加层**。

---

## 1. 技术规格（必须与程序对齐）

| 项目 | 数值 | 说明 |
|------|------|------|
| 单格尺寸 `TILE` | **40×40**（屏幕像素，逻辑世界单位与像素 1:1） | 离屏大地图画布 **6400×4800** |
| 格网尺寸 | **160×120** 格 | `WORLD.w / TILE` × `WORLD.h / TILE` |
| 地图数据 | **3 类枚举** | `T_VOID`、`T_ROAD`、`T_BUILD`；无独立“布景格”ID，布景楼与设施块在格网上 **均为 `T_BUILD`**（见 §3） |
| 可走规则 | 仅 **`T_ROAD`** | 实体圆形半径 16，按覆盖到的格子聚合判定 |

**交付建议**：每种基底格提供 **40×40** 基准图；若需高清再提供 **80×80 / 120×120**（缩放 0.5 / 0.33 使用），并保证 **边缘可无缝平铺**（或与 autotiling 规则一致）。

## 2. 三类基底格（当前程序配色 → 美术方向）

下列 RGB 来自 `ensureTileTerrainCanvas()`，是现行视觉基调，可保留色相并加强材质细节。

### 2.1 `T_VOID` — 野地 / 非路面 / 不可走基底

- **程序色**：`RGB(14, 22, 36)` — 极深蓝灰
- **叙事语义**：岸线基地外的荒地、旷地、未铺装区域；西岸在玩法上与 **岸线侵入** 渐变叠加（见 §5），底色宜偏冷，便于叠青紫光带仍可读。
- **美术关键词**：风化地表、砂砾与硬土、零星低矮植被剪影（避免细节过小在 40px 下糊成一团）、远处工业残骸 **弱暗示** 可点缀；整体 **低对比、无强烈轮廓线**，与路面区分明显。

### 2.2 `T_ROAD` — 可走路面

- **程序色**：`RGB(48, 54, 68)` — 冷灰铺装
- **叙事语义**：基地路网（含主街、支路、斜路段在格网上的投影）；逻辑上路宽不等，但 **格属性只有“路/非路”**，不要求一张图对应矢量路宽。
- **美术关键词**：硬化路面、修补裂缝、淡淡车辙或导向刻痕（勿过亮）、**中灰略偏蓝**；可准备 **轻噪点** 避免纯色块。
- **扩展（可选，利于后续 autotile）**：
  - 直线（横 / 竖）、直角弯、三岔、十字路口、斜向路段边缘模糊过渡（与 void 的 **软边** 或 **路缘石** 二选一风格统一即可）。

### 2.3 `T_BUILD` — 建筑占格 / 阻挡

- **程序色**：`RGB(26, 32, 44)` — 更深蓝灰块
- **语义涵盖**：**设施核心 `FACILITIES` 的占地** + **沿街布景楼 `DECOR_BUILDINGS`**；格网层 **不区分** 二者，画面上层会再画 **设施 POI 半透明矩形与中文名**（见 §4）。
- **美术关键词**：抽象屋面/体量剪影、天窗高亮 **极弱**、竖向条状幕墙暗示；**勿**在 40px 内画可辨 doors/windows 剧情特异细节（避免与具体设施绑定）；整体比 `T_ROAD` **更深、略冷**。

---

## 3. 与程序逻辑相关的注意点（避免美术与碰撞脱节）

1. **只有三种贴图 ID**：若将来要 **“设施 / 布景”** 不同材质，需程序侧增加第四枚举或 mask；当前文档仍以三态为准。
2. **路格已做几何加宽**：部分路面在视觉上可能 **2～3 格宽**，但每格仍用同一 `T_ROAD` 贴图即可；若要 **路肩/中央分道线**，建议仅做 **连续纹理** 或 **随机偏移**，不要依赖单格语义。
3. **格心判定建筑**：设施 core 与格网对齐时，可能存在 **贴建筑边的一列仍为路格**——路面贴图应避免出现“半截建筑”的强暗示，以免与上层 POI 框错位。

## 4. 非瓦片叠加层（可选用统一画风参考）

下列元素 **不是** 独立 tile 文件，但会与格网同屏；AI 批量出图时可当作 **同一世界观下的 UI/特效** 参考色。

### 4.1 格网线

- **样式**：`rgba(0,0,0,0.14)` 1px；主世界与小地图缩略格网均基于此离屏层。

### 4.2 设施 POI（`FACILITIES`）

- **体块填充**：`rgba(18, 24, 34, 0.35)`
- **描边**：`rgba(160, 200, 255, 0.45)`，约 2px
- **中文名**：`rgba(220, 235, 255, 0.96)`，粗体

### 4.3 探索分区 `explorer_zones`（后端可下发）

- **非阻挡区**：虚线描边 `rgba(110, 255, 170, 0.22)`
- **阻挡区** 按 `lock_type`：剧情橙 / 信息紫 / 能力蓝 / 资源黄 / 时间蓝等（浅填充 + 虚线描边，见代码内 `lockedStroke` / `lockedFill`）

### 4.4 实体图标（非 tile）

- **NPC**：暖橙圆点 `rgba(255, 200, 120, 0.95)` 等
- **玩家**：青绿圆 `rgba(120, 255, 200, 0.95)`；小地图 `#6fffc2`
- **任务指引三角**：`rgba(90, 255, 190, 0.95)`

## 5. 岸线侵入（全图渐变层，非单格资源）

`drawShoreIntrusion` / 小地图西岸渐变：青—紫—深蓝 **自西向东衰减**，带轻微 **扫描/波纹亮线**（`lighter` 混合）。  
若单独出 **特效序列** 或 **Shader 参考**：关键词 * shoreline sci-fi corruption / cyan-magenta gradient / subtle scanlines *；**不需要** 为每一格导出独立 PNG，除非要做 **void 格上的局部污染 variant**（与 `T_VOID` 同尺寸可切换）。

## 7. 世界观关键词（统一 briefing）

**源纪元 · 岸线基地**：近未来、偏冷色调、海岸威胁与秘密设施；大地图为 **自上而下** 或 **等轴可选**（当前程序为纯 2D 正交），避免过于卡通的高饱和配色；路面与建筑块宜 **低饱和、高明度差仅限于路/非路**。

---

## 8. 清单速查（给 AI 的一句 prompt 骨架）

> 40×40 seamless top-down game tiles, cold sci-fi coastal base: (1) dark blue-gray wasteland void tile RGB~14,22,36 (2) cool gray paved road tile RGB~48,54,68 with subtle wear (3) darker building roof mass tile RGB~26,32,44 abstract readable as urban blocks; no text, no characters, tileable edges, low saturation.

如需 **中文生图模型**，可将 §2 各小节「美术关键词」与 §7 合并为短描述，并反复强调 **无缝平铺、40px、俯视、无文字**。

## 9. 设施贴图与 `core` 占地对照表

数据来自 **`web/explorer/main.js`** 中 `FACILITIES[].core`（世界坐标像素，与画布 1:1）。贴图可按 **`core.w` × `core.h`** 出 **1×** 逻辑图；高清建议 **@2×** 即 **`(core.w×2)` × `(core.h×2)`** 像素 PNG/WebP。

**`id` 与 `game/narrative_map.py` API 一致**，推荐文件名：`facility_<id>_<w>x<h>.png`（尺寸为逻辑宽高）。
