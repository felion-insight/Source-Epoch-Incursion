面给出一个轻量级、适合网页2D点对点地图的 NPC 游荡方案。

一、基础规则
地图结构：采用“地点-连接”的点对点模式（如前文地图设计），每个地点是一个独立的场景（背景图 + 可交互点）。

NPC 游荡：每个 NPC 有 1 个出生点（初始位置）和 1 个活动范围（可前往的地点列表）。他们会在这些地点之间按照一定逻辑移动。

玩家感知：玩家打开地图界面时，可以看到每个地点上当前有哪些 NPC 的图标（可选）。当玩家进入某个地点，该地点的 NPC 会出现在场景中，可对话。

性能：NPC 位置状态只需简单内存变量（string 类型记录当前所在地点），无需寻路或多帧移动动画（因为点对点移动是瞬间切换场景，NPC 移动不需要表现过程）。

二、各 NPC 游荡方案
1. 卡伦（安全官）
职责：安保、巡逻、监控。

出生点：基地核心（办公室）。

活动范围：基地核心、海岸防线、通讯阵列。

移动逻辑：

白天（游戏内 08:00-20:00）：在基地核心和通讯阵列之间随机切换，每 2 小时（游戏时间）移动一次。

夜晚（20:00-08:00）：固定前往海岸防线巡视。

特殊事件触发：岸线侵入警报时，强制移动到海岸防线。

玩家影响：如果玩家在某个地点找到卡伦，可以触发与地点相关的特殊对话（例如在海岸防线：“我在检查防御工事，没空闲聊。”）

2. 林博士
职责：科研、实验、数据分析。

出生点：医疗实验室。

活动范围：医疗实验室、基地核心、地下监听站（若已建造）。

移动逻辑：

上午（08:00-12:00）：固定在医疗实验室。

下午（12:00-18:00）：前往基地核心会议室讨论数据。

晚上（18:00-22:00）：若地下监听站已存在，则前往监听站；否则回实验室。

深夜（22:00-08:00）：回实验室休息（不可对话）。

特殊：如果玩家在某个地点发现林博士，他可能会主动谈论在该地点的研究（如在监听站：“这里的信号很异常……”）

3. 小胖
职责：维修、物资搬运、偷摸照顾弟弟。

出生点：源矿采集点。

活动范围：源矿采集点、基地核心、废弃矿场表层、地下实验室（隐藏，完成支线前不显示）。

移动逻辑：

白天：随机在源矿采集点和基地核心之间移动（维修任务）。

傍晚（18:00-20:00）：固定出现在废弃矿场表层（搬运物资）。

夜晚（20:00-06:00）：如果地下实验室未暴露，他会偷偷前往地下实验室（玩家需要有跟踪选项才能发现）；如果支线已完成，他偶尔会去实验室看望弟弟。

特殊：小胖的移动轨迹可能会暴露隐藏地点（玩家跟随他可发现地下实验室入口）。

4. 堇（净空会特工）
职责：生态研究（伪装），实际收集情报。

出生点：基地核心（生态研究室）。

活动范围：基地核心、海岸线洞穴、净空会圣树（后期解锁）。

移动逻辑：

初期（第一幕）：只在基地核心出现。

中期（第二幕）：随机前往海岸线洞穴（收集样本）。

后期（第三幕，若玩家未敌对）：可能前往净空会圣树（需解锁该地点）。

特殊：堇出现在非基地地点时，玩家与她对话可推进净空会支线。

5. 回声-7
职责：AI 联络，无物理移动，但可“出现”在任何有电子设备的区域。

出现规则：

在通讯阵列、基地核心（有屏幕的房间）、地下监听站、医疗实验室（电脑终端）等地点，玩家有概率（30%）遇到回声-7 弹出文字消息。

如果玩家安装了加密模块或接受了援助，概率提升至 80%。

无需出生点，只要玩家到达上述地点，即可能触发。

6. 克莱因
固定位置：沉没实验室（深层囚室）。不游荡。

解锁后：只要玩家进入该地点，克莱因就在那里（濒死状态）。

7. 源
无移动，只在特定地点（地下监听站、高同调值梦境）通过幻觉/低语出现。

三、实现技术建议（极简）
数据存储
javascript
// 每个NPC的当前位置
npcLocations = {
  "Karen": "BaseCore",
  "DrLin": "MedLab",
  "Chubby": "SourceMine",
  "Jin": "BaseCore"
}

// 每个NPC的移动计划（基于游戏内计时器）
npcSchedules = {
  "Karen": [
    { timeRange: [8,20], locations: ["BaseCore","CommsArray"], interval: 2 },
    { timeRange: [20,8], locations: ["CoastLine"], fixed: true }
  ]
}
更新逻辑
使用游戏内的全局时钟（小时为单位）。每经过 1 小时（或者玩家移动一次），触发所有 NPC 根据当前时间重新计算位置。

简单版：仅在玩家进入某个地点时，重新计算所有 NPC 的当前位置（根据上次更新时间快进模拟）。这样可以避免后台定时器，降低复杂度。

玩家界面显示
在地图界面的每个地点图标旁，用小型头像图标显示当前有哪些 NPC 在该地点。

玩家点击地点进入后，场景中就能看到这些 NPC 并与之对话。

四、可能带来的设计收益
沉浸感提升：玩家发现 NPC 不在固定位置，需要“去找他们”，更像真实的人际关系。

隐藏内容引导：跟随小胖可发现地下实验室；在特定地点遇到林博士可触发额外科研对话。

重复可玩性：不同时间、不同路线遇到的 NPC 位置不同，可能触发不同的临时对话。

五、需要注意的风险
玩家找不到关键 NPC：为了避免卡关，关键剧情触发时（如需要向卡伦交任务），该 NPC 应该强制回到固定位置（例如任务期间卡伦固定在办公室）。可以在任务日志中提示“卡伦通常在海岸防线或办公室”。

性能开销：点对点地图 + 时间戳模拟，几乎没有性能问题。
